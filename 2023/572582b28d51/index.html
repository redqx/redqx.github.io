Windows API进内核的的方式

2023-12-15 blog

转自 https://blog.csdn.net/qq_40363731/article/details/130095893

在 Windows操作系统 中，应用程序要与内核进行交互时，需要通过一系列的步骤来实现。

这些步骤包括调用应用程序接口（API），转到内核模式，然后执行内核函数。

在本文中，我们将详细介绍Windows API进入内核的流程，以及在不同架构的系统中这一流程是如何实现的

首先我们来了解一下Windows API进入内核的基本流程：

应用程序调用API，例如OpenProcess。

API通过跳转指令（jmp）将调用转发给相应的DLL文件，如kernel32.dll。

DLL文件继续将调用转发给内核基础库，如kernelbase.dll，这里的函数通常以Zw或Nt为前缀。

最后，调用到达ntdll.dll，并通过特定的指令进入内核模式，例如sysenter、syscall或int 2E。

接下来，我们将详细讨论在不同架构的操作系统和应用程序中，这一流程是如何实现的。

ps: 这里我们提到了kernelbase.dll

接下来，我们将详细讨论在不同架构的操作系统和应用程序中，这一流程是如何实现的。

我们以OpenProcess为例

一、x86应用程序

x86应用程序在x86操作系统上：

exe -> OpenProcess

进入 kernel32.dll -> jmp

进入 kernelbase.dll -> ZwOpenProcess

进入 ntdll.dll -> sysenter

x86应用程序在x64操作系统上：

exe -> OpenProcess

进入 kernel32.dll -> jmp

进入 kernelbase.dll -> NtOpenProcess

进入 ntdll.dll -> Wow64

二、x64应用程序

x64应用程序在x64操作系统上：

exe -> OpenProcess

进入 kernel32.dll -> jmp

进入 kernelbase.dll -> NtOpenProcess

进入 ntdll.dll -> syscall/int 2E
